# Architecture

Reconstruction of the network architecture described in Waqar et al., *Sensors*
2025, 25(18), 5830 (Section 2.3-2.5).

## Overview

The framework (`src/models/network.py::DualSegmentationFramework`) is a two-stage,
sequentially-trained pipeline plus a classical post-processing operator:

```
RGB image ─► [Stage 1] RTMDet-ins-m ─► per-shrimp instance masks
                                          │
            RGB + instance mask ─► [Stage 2] Centerline Segmentation Model ─► centerline mask
                                          │
                    Zhang–Suen thinning ─► 1-px centerline ─► arc length (px) ─► size (mm)
```

## Stage 1 — Instance segmentation (RTMDet-ins-m)

Module: `src/models/network.py::RTMDetIns` (Equations 1-2).

| Component | Implementation | File |
|-----------|----------------|------|
| Backbone  | CSPNeXt (`deepen=0.67`, `widen=0.75`) with CSP layers, channel attention and an SPPF block | `encoder.py::CSPNeXt` |
| Neck      | CSPNeXtPAFPN (top-down + bottom-up path aggregation) | `neck.py::CSPNeXtPAFPN` |
| Head      | Decoupled classification / box-distance / kernel branches | `head.py::RTMDetInsHead` |
| Mask      | Shared mask-prototype branch + per-instance dynamic convolution | `head.py::MaskFeatModule`, `network.py::RTMDetIns._dynamic_masks` |

Given an image `I ∈ R^{H×W×3}`, the backbone produces multi-scale features
`F = {F_l}` (Eq. 1). The decoupled head predicts class logits, distance-encoded
boxes and a vector of dynamic-convolution parameters per location; instance masks
are generated by convolving shared prototype features (augmented with relative
coordinates) with the per-instance dynamic kernels `M̂_i = H_dyn(P_mask, θ_i)`
(Eq. 2).

> **Reconstruction note.** The paper trains RTMDet with the official OpenMMLab
> recipe (dynamic soft-label assignment). The bundled `RTMDetIns` faithfully
> mirrors the *architecture* and exposes the losses (`RTMDetLoss`); for exact AP
> reproduction, train via the OpenMMLab RTMDet config on the COCO-format dataset
> produced by `src/datasets`. The `predict()` decode is sufficient to run the
> full pipeline end-to-end.

## Stage 2 — Centerline segmentation (proposed model)

Module: `src/models/network.py::CenterlineSegmentationModel` (Section 2.5.1,
Equations 4-6). It enhances DeepLabv3 by fusing ASPP multi-scale context with a
U-Net encoder-decoder for fine spatial detail.

1. **Input fusion (Eqs. 4-5).** Concatenate the RGB image with the binary instance
   mask → 4 channels, then a `1×1` convolution projects back to 3 channels
   (`fusion.py::InputFusion`). This injects object-location priors into the input.
2. **Parallel branches.** The fused image feeds (a) a MobileNetV2 U-Net encoder
   (`encoder.py::MobileNetV2Encoder`, strides 2/4/8/16/32) and (b) an ASPP context
   module (`aspp.py::ASPP`, dilation rates 6/12/18). The deepest encoder feature is
   concatenated with a downsampled fused image to guide ASPP with extra spatial cues.
3. **Decoder.** The ASPP output is upsampled and fused with the U-Net skip
   connections, each stage followed by convolutional refinement
   (`decoder.py::UNetDecoder`, `fusion.py::FeatureFusion`).
4. **Head (Eq. 6).** A final convolution + sigmoid yields the centerline mask
   `Ŷ = σ(C_final(U(F_dec)))` (`head.py::SegmentationHead`).

The ground-truth centerline is rasterised as a **3-pixel-wide** band during
training to improve gradient flow.

## Stage 3 — Size estimation

Module: `src/utils/skeletonize.py` (Eq. 9) and `src/utils/geometry.py`
(Eqs. 10-11). The Zhang–Suen thinning algorithm reduces the predicted centerline
to a one-pixel-wide medial axis; its arc length in pixels is converted to
millimetres with a calibration factor obtained from a reference object.
